# DESIGN.md

# 스쿨메이트 AI 디자인 문서
## Apple-inspired Minimal Work Dashboard

---

## 1. 디자인 방향

스쿨메이트 AI는 학교 업무 웹앱이지만, 시각적으로는 복잡한 행정 시스템처럼 보이지 않아야 한다.

디자인 방향은 **Apple식 미니멀리즘을 교사용 업무 앱에 맞게 재해석**하는 것이다.

Apple 스타일에서 가져올 요소는 다음과 같다.

- 단일 액션 컬러
- 넓은 여백
- 조용한 타이포그래피
- 카드 그림자 최소화
- pill 형태 버튼
- frosted 상단바
- 기능보다 핵심 콘텐츠가 먼저 보이는 화면
- 장식보다 정보의 명확성

단, 스쿨메이트는 제품 사진 중심의 마케팅 페이지가 아니라 업무 처리 앱이므로, Apple의 분위기를 그대로 복사하지 않고 **업무 카드가 주인공이 되는 조용한 SaaS 화면**으로 적용한다.

---

## 2. 핵심 UX 원칙

### 2.1 기능은 많지만 첫 화면은 단순하게

홈 화면에는 아래 3가지만 보여준다.

1. 오늘 할 일
2. AI 업무 후보
3. 빠른 실행

표, 전체 달력, 전체 예약표, 관리자 설정은 숨긴다.

---

### 2.2 업무 카드가 주인공

Apple 사이트에서 제품 이미지가 주인공이라면, 스쿨메이트에서는 **교사가 오늘 처리해야 할 업무 카드**가 주인공이다.

따라서 디자인은 업무카드를 방해하지 않아야 한다.

- 장식 아이콘 과다 사용 금지
- 과한 그림자 금지
- 복잡한 색상 배지 금지
- 첫 화면 정보량 제한
- 클릭 후 상세 정보 표시

---

### 2.3 AI 결과는 조심스럽게 표시

AI가 추출한 업무는 확정 정보가 아니라 후보 정보다.

따라서 UI 문구도 다음처럼 사용한다.

| 좋지 않은 표현 | 권장 표현 |
|---|---|
| AI가 등록한 일정 | AI가 찾은 일정 후보 |
| 자동 업무 생성 | 업무 후보 생성 |
| 생기부 작성 | 세특 문구 점검 |
| AI 판단 완료 | 교사 확인 필요 |

## 3. 색상 시스템

### 3.1 기본 색상

| 용도 | 색상 | 값 |
|---|---|---|
| Primary Action Blue | 주요 버튼, 링크, 선택 상태 | `#0066CC` |
| Primary Hover Blue | 버튼 hover/focus | `#0071E3` |
| Link on Dark | 어두운 배경의 링크 | `#2997FF` |
| Ink | 주요 텍스트 | `#1D1D1F` |
| Muted Ink | 보조 텍스트 | `#6E6E73` |
| Canvas | 기본 배경 | `#FFFFFF` |
| Parchment | 부드러운 보조 배경 | `#F5F5F7` |
| Pearl Surface | 카드 배경 | `#FAFAFC` |
| Hairline | 얇은 구분선 | `#E0E0E0` |
| Soft Divider | 부드러운 구분선 | `#F0F0F0` |

### 3.2 상태 색상

Apple 스타일의 단일 블루 원칙을 유지하되, 학교 업무 앱에서는 상태 구분이 필요하다.  
따라서 상태색은 최소한만 사용한다.

| 상태 | 색상 | 사용 위치 |
|---|---|---|
| 일반 | Blue | 기본 액션 |
| 중요 | Orange | 중요 일정, 마감 임박 |
| 긴급 | Red | 즉시 확인 필요 |
| 완료 | Green | 완료 체크 |
| 비활성 | Gray | 비활성 버튼, 흐린 정보 |

상태색은 넓은 면적 배경으로 사용하지 않고, 작은 점, 라벨, 배지에만 사용한다.

## 4. 타이포그래피

### 4.1 폰트 패밀리

```css
font-family: "SF Pro Text", "SF Pro Display", system-ui, -apple-system, BlinkMacSystemFont, "Inter", sans-serif;
```

Windows 환경에서는 Inter 또는 system-ui로 대체한다.

### 4.2 타이포그래피 토큰

| 토큰 | 크기 | 굵기 | 줄간격 | 용도 |
|---|---:|---:|---:|---|
| hero | 40px | 600 | 1.1 | 로그인/랜딩 메인 문구 |
| display | 32px | 600 | 1.2 | 주요 페이지 제목 |
| title | 24px | 600 | 1.3 | 카드 그룹 제목 |
| section | 21px | 600 | 1.3 | 섹션 제목 |
| body | 17px | 400 | 1.47 | 본문 |
| body-strong | 17px | 600 | 1.35 | 강조 본문 |
| caption | 14px | 400 | 1.43 | 보조 설명 |
| fine | 12px | 400 | 1.3 | 라벨, 메타 정보 |

### 4.3 타이포그래피 원칙

- 제목은 굵지만 과하지 않게 600을 사용한다.
- 본문은 17px을 기본으로 하여 읽기 편하게 한다.
- 카드 안의 설명은 짧게 유지한다.
- 모바일에서는 제목 크기를 줄이되, 버튼과 본문은 읽기 편한 크기를 유지한다.

## 5. 레이아웃

### 5.1 PC 레이아웃

```text
상단바
[학교명] [검색] [Google Chat 연결 상태] [알림] [프로필]

왼쪽 메뉴
홈 / 업무함 / 일정 / 특별실 / 학생기록 / 설정

메인
[오늘 할 일]
[AI 업무 후보]
[빠른 실행]
```

### 5.2 모바일 레이아웃

```text
상단
[스쿨메이트 AI] [알림] [프로필]

메인
[오늘 할 일]
[AI 업무 후보]
[빠른 실행]

하단 탭
홈 / 업무함 / 일정 / 예약 / 기록
```

### 5.3 간단 보기

간단 보기는 화면을 최대한 줄인다.

표시 항목:

- 오늘 할 일 최대 3개
- AI 업무 후보 최대 2개
- 빠른 실행 최대 3개
- 알림 배지

## 6. 홈 화면 디자인

### 6.1 홈 화면 원칙

홈 화면은 “업무 시작 화면”이다.  
관리 화면이나 전체 현황판이 아니다.

### 6.2 홈 화면 구성

| 영역 | 표시 개수 | 설명 |
|---|---:|---|
| 오늘 할 일 | 최대 3개 | 오늘 처리할 개인/부서 업무 |
| AI 업무 후보 | 최대 2개 | Google Chat에서 감지된 업무 후보 |
| 빠른 실행 | 최대 3개 | 새 업무, 일정, 문구 점검 등 |
| 알림 | 배지/팝업 | 필요한 때만 표시 |

### 6.3 홈 화면에서 숨기는 것

- 전체 특별실 예약표
- 전체 일정 달력
- 학생기록 전체 자료 목록
- 관리자 설정
- Google Chat 세부 연동 설정
- AI 분석 로그

---

## 7. 컴포넌트 디자인

### 7.1 상단바

Apple의 frosted sub-nav 느낌을 적용한다.

| 속성 | 값 |
|---|---|
| 높이 | 56px |
| 배경 | `rgba(245,245,247,0.82)` |
| 효과 | `backdrop-filter: saturate(180%) blur(20px)` |
| 구분선 | 1px hairline |
| 텍스트 | 14px 또는 17px |

상단바에는 많은 버튼을 넣지 않는다.

포함 요소:

- 학교명
- 검색
- Google Chat 연결 상태
- 알림
- 프로필

---

### 7.2 왼쪽 메뉴

PC에서만 고정 사이드바를 사용한다.

| 항목 | 원칙 |
|---|---|
| 메뉴 수 | 6개 이하 |
| 아이콘 | 단순 라인 아이콘 |
| 색상 | 선택 메뉴만 Primary Blue |
| 배경 | 흰색 또는 parchment |
| 그림자 | 사용하지 않음 |

메뉴:

- 홈
- 업무함
- 일정
- 특별실
- 학생기록
- 설정

---

### 7.3 하단 탭

모바일에서는 하단 탭을 사용한다.

탭:

- 홈
- 업무함
- 일정
- 예약
- 기록

설정은 프로필 메뉴 안에 둔다.

---

### 7.4 카드

카드는 그림자보다 얇은 선과 여백으로 구분한다.

| 속성 | 값 |
|---|---|
| 배경 | `#FFFFFF` |
| Border | `1px solid #E0E0E0` |
| Radius | 18px |
| Padding | 24px |
| Shadow | 기본 사용하지 않음 |
| Hover | border 색상 또는 배경 변화만 |

카드에 많은 정보를 넣지 않는다.

---

### 7.5 버튼

#### Primary Button

```css
background: #0066CC;
color: #FFFFFF;
border-radius: 9999px;
padding: 11px 22px;
font-size: 17px;
font-weight: 400;
```

#### Secondary Button

```css
background: #FFFFFF;
color: #0066CC;
border: 1px solid #0066CC;
border-radius: 9999px;
padding: 11px 22px;
```

#### Utility Button

```css
background: #F5F5F7;
color: #1D1D1F;
border-radius: 11px;
padding: 8px 14px;
font-size: 14px;
```

버튼 active 상태는 `transform: scale(0.97)`을 사용한다.

---

### 7.6 알림

알림은 항상 펼쳐두지 않는다.

| 환경 | 표시 방식 |
|---|---|
| PC | 알림 아이콘 배지 + 드롭다운 |
| 모바일 | 토스트 + 바텀시트 |
| 간단 보기 | 중요 알림만 표시 |

---

### 7.7 모달과 바텀시트

- PC 상세 정보는 오른쪽 패널 또는 모달로 표시한다.
- 모바일 상세 정보는 아래에서 올라오는 바텀시트로 표시한다.
- 일정 등록, 예약, 문구 점검은 전체 화면 전환보다 모달을 우선한다.

---

## 8. 업무함 디자인

### 8.1 업무함 구조

업무함은 Google Chat 전체 메시지를 보여주는 곳이 아니다.  
AI가 업무 후보로 판단한 메시지를 우선 보여준다.

| 탭 | 설명 |
|---|---|
| 업무 후보 | AI가 업무로 판단한 메시지 |
| 저장한 업무 | 교사가 저장한 업무 |
| 무시한 메시지 | 사용자가 무시한 메시지 |
| 원문 보기 | 필요한 경우만 원문 확인 |

### 8.2 AI 업무 후보 카드

카드에는 아래 정보만 우선 표시한다.

- 업무 후보 제목
- 출처 Space
- 날짜 또는 확인 필요
- 중요도
- 버튼: 저장 / 일정 / 무시

상세 분석은 카드 클릭 시 표시한다.

## 9. 일정 디자인

### 9.1 PC 일정

PC에서는 월간 달력과 리스트를 함께 제공하되, 홈에는 노출하지 않는다.

### 9.2 모바일 일정

모바일에서는 월간 달력보다 리스트를 우선한다.

```text
오늘
- 10:30 학년부 회의
- 14:00 체험학습 안내 확인

이번 주
- 7월 3일 체험학습
- 7월 5일 평가 마감
```

---

## 10. 특별실 예약 디자인

### 10.1 PC

PC에서는 날짜·교시·특별실 기준 표를 사용할 수 있다.

### 10.2 모바일

모바일에서는 표 대신 리스트를 사용한다.

```text
7월 3일

컴퓨터실
1교시 예약 가능
2교시 예약됨
3교시 예약 가능
```

### 10.3 첫 화면 노출 금지

특별실 예약률, 예약완료 건수, 예약대기 건수, 예약불가 건수는 홈 화면에 노출하지 않는다.

---

## 11. 학생기록 디자인

### 11.1 메뉴명

왼쪽 메뉴에서는 **학생기록**으로 짧게 표시한다.  
화면 안 제목은 **학생기록 도우미**로 표시한다.

### 11.2 탭 구조

- 내 자료실
- 세특 점검
- 기록 템플릿
- 지침 파일
- 활동 메모

### 11.3 민감성 완화 UI

세특 점검 화면에는 안내 문구를 고정한다.

> AI는 생기부를 대신 작성하지 않습니다. 교사가 작성한 문구를 점검하고 개선 의견을 제안하는 보조 도구입니다.

## 12. 반응형 기준

| 화면 너비 | 적용 |
|---|---|
| 1200px 이상 | PC 대시보드 |
| 768px~1199px | 태블릿 카드형 |
| 767px 이하 | 모바일 하단 탭 |
| 사용자 고정 | 사용자가 선택한 보기 우선 |

### 모바일 숨김 원칙

- 사이드바 숨김
- 대형 달력 숨김
- 표는 리스트로 변환
- 상세 분석은 바텀시트
- 설정은 프로필 안으로 이동

---

## 13. Do & Don't

### Do

- 단일 Primary Blue를 기본 액션 컬러로 사용한다.
- 첫 화면은 최대 3개 영역만 유지한다.
- 카드에는 최소 정보만 담는다.
- 상세 기능은 클릭 후 보여준다.
- 모바일에서는 리스트와 바텀시트를 사용한다.
- AI 결과에는 항상 “후보”, “확인 필요” 표현을 사용한다.

### Don't

- 첫 화면에 모든 위젯을 나열하지 않는다.
- 카드에 과한 그림자를 넣지 않는다.
- 색상 배지를 남발하지 않는다.
- AI가 자동으로 등록한 것처럼 표현하지 않는다.
- 생기부 작성 대체처럼 보이는 표현을 사용하지 않는다.
- 모바일에 복잡한 표를 그대로 넣지 않는다.

---

## 14. 디자인 핵심 문장

스쿨메이트 AI는 기능을 많이 보여주는 업무 포털이 아니라, 교사가 오늘 처리해야 할 업무를 조용히 드러내는 미니멀 업무 시작 화면이다.  
Apple의 넓은 여백, 단일 블루 액션, 조용한 타이포그래피를 적용하되, 학교 업무 앱의 정보성과 처리 속도를 잃지 않도록 설계한다.
