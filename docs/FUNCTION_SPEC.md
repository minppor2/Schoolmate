# 스쿨메이트 AI 기능 명세서

---

## 1. 기능 개요

스쿨메이트 AI는 Google Chat 업무방과 연동하여 메시지 속 업무를 AI가 후보로 추출하고, 교사가 확인 후 업무카드·일정·공지문·알림으로 전환할 수 있도록 돕는 교사용 업무 자동화 웹앱이다.

핵심 기능은 다음 흐름에 집중한다.

```text
Google Chat 메시지 불러오기
→ AI 업무 후보 생성
→ 교사 확인
→ 업무카드 저장 / 일정 등록 / 공지문 변환
```

## 2. 로그인 및 사용자 설정

### 2.1 기능 설명

사용자는 Google 계정으로 로그인한다.  
사용자의 역할에 따라 접근 가능한 메뉴와 기능이 달라진다.

### 2.2 기능 목록

| 기능 | 설명 |
|---|---|
| Google 로그인 | Firebase Authentication 사용 |
| 사용자 정보 저장 | 이름, 이메일, 역할, 부서 저장 |
| 보기 모드 저장 | 자동/PC/모바일/간단/관리자 보기 |
| 알림 설정 저장 | 사용자별 알림 수신 여부 저장 |
| 홈 위젯 저장 | 사용자별 홈 화면 구성 저장 |

### 2.3 역할

| 역할 | 권한 |
|---|---|
| 일반 교사 | 본인 업무, 일정, 예약, 자료 관리 |
| 담임교사 | 학급 일정, 학생기록 자료 관리 |
| 부장교사 | 부서/학년 업무 확인, 공지 발송 |
| 관리자 | 학교 설정, 권한, Chat Space 연동 |

## 3. 사용자별 화면 모드

### 3.1 모드

| 모드 | 설명 |
|---|---|
| 자동 맞춤 | 화면 크기에 따라 자동 전환 |
| PC 보기 | 넓은 대시보드형 |
| 모바일 보기 | 1열 카드와 하단 탭 |
| 간단 보기 | 오늘 할 일, 업무 후보, 빠른 실행만 표시 |
| 관리자 보기 | 설정, 권한, 연동 관리 표시 |

### 3.2 요구사항

- 기본값은 자동 맞춤이다.
- 일반 사용자에게는 간단 보기 전환만 쉽게 노출한다.
- 관리자 보기는 권한이 있는 사용자만 볼 수 있다.
- 마지막 선택 모드는 Firestore에 저장한다.

## 4. 홈 화면

### 4.1 기능 설명

홈 화면은 업무 시작 화면이다.  
모든 기능을 보여주지 않고 가장 필요한 정보만 표시한다.

### 4.2 표시 제한

| 영역 | 최대 표시 개수 |
|---|---:|
| 오늘 할 일 | 3개 |
| AI 업무 후보 | 2개 |
| 빠른 실행 | 3개 |

### 4.3 오늘 할 일

#### 표시 정보

- 업무 제목
- 시간 또는 마감일
- 중요도
- 완료 체크
- 관련 일정 여부

#### 기능

- 완료 처리
- 상세 보기
- 전체 보기
- 알림 연결

## 5. Google Chat 연동 업무함

### 5.1 기능 설명

Google Chat의 승인된 업무용 Space에서 메시지를 가져와 업무 후보를 생성한다.

### 5.2 핵심 원칙

- 모든 대화 수집 금지
- 개인 DM 수집 금지
- 승인된 업무용 Space만 연동
- 메시지 원문 보관 기간 설정
- AI 결과 자동 실행 금지
- 교사 확인 후 저장

### 5.3 구현 단계

| 단계 | 구현 내용 |
|---|---|
| 1차 | 관리자가 승인한 Space의 최근 메시지 조회 |
| 2차 | Google Workspace Events API로 실시간 감지 |
| 3차 | Chat 메시지에 업무 저장 버튼 제공 |

### 5.4 메시지 필터

AI 분석 전에 기본 필터를 적용한다.

| 필터 | 설명 |
|---|---|
| 기간 | 최근 7일 메시지 우선 |
| Space | 승인된 Space만 |
| 유형 | 업무 키워드 포함 메시지 우선 |
| 제외 | 개인 대화, 잡담성 메시지 제외 |

### 5.5 업무함 탭

| 탭 | 설명 |
|---|---|
| 업무 후보 | AI가 업무로 판단한 메시지 |
| 저장한 업무 | 교사가 저장한 업무 |
| 무시한 메시지 | 사용자가 무시한 메시지 |
| 원문 보기 | 필요한 경우만 원문 확인 |

## 6. AI 업무 추출

### 6.1 기능 설명

Google Chat 메시지를 분석하여 업무 후보를 구조화한다.

### 6.2 추출 항목

| 항목 | 설명 |
|---|---|
| 업무명 | 메시지의 핵심 업무 제목 |
| 날짜 | 일정 또는 마감 날짜 |
| 시간 | 일정 시간 |
| 대상 | 개인, 부서, 학년, 전체 |
| 장소 | 회의실, 특별실, 외부 장소 |
| 해야 할 일 | 처리해야 할 업무 |
| 중요도 | 일반, 중요, 긴급 |
| 신뢰도 | AI 추출 결과 confidence |
| 확인 필요 | 날짜나 대상이 불명확한 경우 |

### 6.3 AI 결과 상태

| 상태 | 설명 |
|---|---|
| candidate | 후보 생성 |
| needsReview | 교사 확인 필요 |
| savedAsTask | 업무카드 저장 완료 |
| savedAsSchedule | 일정 저장 완료 |
| ignored | 사용자가 무시 |
| error | 분석 실패 |

### 6.4 안전 장치

- confidence가 낮으면 “확인 필요”로 표시한다.
- 날짜가 불명확하면 자동 등록하지 않는다.
- 대상이 모호하면 전체 알림을 금지한다.
- 교사 확인 전에는 DB에 확정 업무로 저장하지 않는다.

## 8. 일정 관리

### 8.1 기능 설명

업무 후보 또는 직접 입력으로 일정을 등록·수정·변경·삭제한다.

### 8.2 주요 기능

| 기능 | 설명 |
|---|---|
| 일정 등록 | 새 일정 추가 |
| 일정 수정 | 제목, 시간, 장소 수정 |
| 일정 변경 | 날짜 또는 시간 변경 |
| 일정 삭제 | 일정 삭제 |
| 중요 일정 지정 | 중요 배지 표시 |
| 대상 설정 | 개인, 부서, 학년, 전체, 선택교사 |
| Google Calendar 등록 | 외부 캘린더 연동 |
| 동기화 상태 표시 | Calendar 연동 상태 확인 |

### 8.3 동기화 상태

| 상태 | 설명 |
|---|---|
| pending | 등록 대기 |
| synced | Calendar 연동 완료 |
| failed | Calendar 연동 실패 |
| deleted | 삭제됨 |

### 8.4 안전 장치

- AI 추출 일정은 후보로만 표시한다.
- 등록 전 확인 모달을 띄운다.
- 삭제 전 확인 모달을 띄운다.
- Calendar 연동 실패 시 `syncStatus: failed`로 저장한다.

## 9. 알림

### 9.1 기능 설명

일정, 업무, 예약, AI 후보를 알림으로 제공한다.

### 9.2 알림 대상

| 대상 | 설명 |
|---|---|
| 개인 | 특정 교사 |
| 부서 | 특정 부서 |
| 학년 | 특정 학년 담당 교사 |
| 전체 | 전체 교사 |
| 선택교사 | 지정된 교사 |

### 9.3 알림 유형

| 유형 | 설명 |
|---|---|
| todaySchedule | 오늘 일정 |
| importantSchedule | 중요 일정 |
| deadline | 마감 임박 |
| aiTaskCandidate | 새 업무 후보 |
| reservation | 예약 변경 |
| recordReview | 세특 점검 결과 |
| system | 시스템 알림 |

### 9.4 알림 피로도 방지

- 사용자가 알림 유형별로 끌 수 있다.
- 전체 알림은 관리자만 보낼 수 있다.
- 중요 알림도 대상자를 지정해야 한다.
- 읽은 알림은 자동 숨김 처리한다.

## 11. 학생기록 도우미

### 11.1 기능 설명

학생기록 도우미는 학생 기록 작성에 필요한 자료를 보관하고, 교사가 작성한 세특 문구를 점검받는 보조 공간이다.

AI가 생기부를 대신 작성하지 않는다.

### 11.2 탭 구조

| 탭 | 설명 |
|---|---|
| 내 자료실 | 웹페이지, 링크, 문서 저장 |
| 세특 점검 | 문구 점검 |
| 기록 템플릿 | 교과별 문장 틀 |
| 지침 파일 | 생기부 지침, 평가 기준 |
| 활동 메모 | 학생 활동 관찰 메모 |

### 11.3 개인정보 안전장치

- 실제 학생명 입력을 권장하지 않는다.
- 발표 시연은 가상 데이터 사용
- 원문 저장 여부 선택 가능
- 민감 표현 감지 시 경고
- 공유 범위 설정: 개인/부서/학교

### 11.4 세특 점검 출력

- 구체성
- 과장 표현 여부
- 금지 표현 여부
- 개인정보 위험 여부
- 개선 제안
- byte 수
- 저장 여부 선택

## 12. 공지/문구 변환

### 12.1 기능 설명

### 12.2 변환 유형

| 유형 | 설명 |
|---|---|
| 학생용 | 쉬운 표현 |
| 학부모용 | 정중한 안내문 |
| 교사용 | 업무 공유용 |
| 공식형 | 공문 느낌 |
| 짧은형 | 문자/메신저용 |

### 12.3 안전 장치

- 교사가 확인 후 복사하거나 저장한다.
- 민감한 표현은 경고한다.
